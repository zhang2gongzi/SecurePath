# SecurePath 项目进度汇报

> 汇报日期：2026-05-20
> 项目目标：提升 LLM 代码生成安全性

---

## 一、项目背景

### 1.1 研究动机

- GitHub Copilot 生成的代码 **27.3% 存在安全漏洞**（Fu et al., ACM TOSEM 2025）
- 涉及 43 种 CWE 类型，其中 8 种属于 2023 CWE Top-25
- 当前 CoT 路径选择方法（如 Self-Consistency）只关注答案频率，**不考虑安全性**

### 1.2 研究问题

**核心问题**：如何让 LLM 在生成多条候选代码时，自动选择最安全的版本？

### 1.3 研究思路

```
用户代码请求 → LLM生成K条候选代码 → 安全编码器评估 → 选择最安全代码
```

## 二、技术路线

### 阶段一：安全感知编码器训练

| 步骤 | 内容 |
|------|------|
| 数据 | 漏洞代码 vs 安全代码 |
| 方法 | 对比学习（CodeBERT + InfoNCE Loss） |
| 目标 | 漏洞代码聚集，安全代码聚集，两者分离 |
| 输出 | 编码器 + 漏洞原型 + 安全原型 |

### 阶段二：CoT 路径安全选择

| 步骤 | 内容 |
|------|------|
| 输入 | 用户代码生成请求 |
| LLM采样 | 生成 K 条推理路径，每条输出一段代码 |
| 安全评估 | 编码器评估每段代码安全性 |
| 输出 | 选择安全得分最高的代码 |

## 四、关键问题分析与解决

### 4.1 问题现象

| 指标 | 结果 | 期望 |
|------|------|------|
| 训练准确率 | 52% | >70% |
| t-SNE 分离分数 | 0.082 | >1.5 |
| 特征可分性 | 几乎不可分 | 明显分离 |

### 4.2 根因分析

**数据集结构深入分析**：

MSR_data_cleaned 数据集中有两个关键字段：
- `func_before`：漏洞代码
- `func_after`：修复代码
- `vul`：0 或 1

**关键发现**：

| vul字段 | 含义 | 记录数 | func_before vs func_after |
|---------|------|--------|---------------------------|
| **vul=0** | 安全代码记录 | 177,736 | **92.7% 完全相同** |
| **vul=1** | 漏洞代码记录 | 10,900 | 有差异 |

**问题本质**：

```
vul=0 记录：安全代码，不需要修复 → func_before == func_after（同一函数）
vul=1 记录：漏洞代码，已修复   → func_before ≠ func_after（修复前后）
```

**原预处理方案的问题**：

```python
# 原方案：混合采样 vul=0 和 vul=1
for _, row in df.iterrows():
    samples.append({'code': row['func_before'], 'label': 1})  # "漏洞"
    samples.append({'code': row['func_after'], 'label': 0})   # "安全"
```

结果：
- 92.7% 的训练样本中，"漏洞代码"和"安全代码"**完全相同**
- 模型被迫学习：**同一代码既是漏洞又是安全** → 自相矛盾
- 这就是特征坍塌、准确率仅 52% 的根本原因

### 4.3 解决方案：方案二

**核心思想**：漏洞代码和安全代码必须来自**不同函数**，语义差异明显

**数据重构**：

```
漏洞代码池：只取 vul=1 的 func_before（真正的漏洞代码）
安全代码池：vul=0 的 func_before + vul=1 的 func_after（来自不同函数）
```

**数据池统计**：

| 数据池 | 来源 | 数量 |
|--------|------|------|
| 漏洞代码池 | vul=1 的 func_before | 7,901 |
| 安全代码池 | vul=0 的 func_before | 111,087 |
| 安全代码池 | vul=1 的 func_after | 7,901 |

**采样结果**：

| CWE类型 | 漏洞采样 | 安全采样 |
|---------|----------|----------|
| CWE-119 | 500 | 500 |
| CWE-416 | 254 | 500 |
| CWE-125 | 456 | 500 |
| CWE-476 | 180 | 500 |
| CWE-190 | 266 | 500 |
| CWE-787 | 166 | 500 |
| **总计** | **1,822** | **3,000** |

**最终数据集**：

| 数据集 | 总计 | 漏洞 | 安全 |
|--------|------|------|------|
| 训练集 | 3,375 | 1,275 | 2,100 |
